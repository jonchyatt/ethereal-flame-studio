<objective>
## Goal
Fix 3 chat/TTS bugs: (1) chat stays open when spotlight appears, (2) chat doesn't scroll to bottom on open, (3) toggling speaker causes double TTS or restarts narration from beginning.

## Purpose
These bugs make the tutorial feel broken. Spotlight + open chat = overlapping UIs. Double TTS = disorienting. These are polish fixes that make the tutorial experience smooth enough for the Jenny demo.

## Root Causes (from code)
1. ChatOverlay has no logic to close when spotlight activates — both UIs show simultaneously
2. Chat scroll: `animState === 'open'` + 50ms timeout may not fire after re-opens with existing messages; `messagesEndRef.scrollIntoView` on the messages-change effect only fires when messages CHANGE, not on open
3. Double TTS: SpotlightOverlay and ChatOverlay each manage independent Audio objects. When `isNarrationEnabled` toggles ON, SpotlightOverlay re-runs its narration effect AND ChatOverlay re-runs its TTS effect for last unspoken message. Both play simultaneously. Also: toggling OFF then ON restarts spotlight narration from beginning (effect dependency on `isNarrationEnabled`).

## Output
- Chat auto-closes when spotlight activates
- Chat always scrolled to bottom when opened
- Only one TTS voice plays at a time (shared singleton audio manager)
</objective>
