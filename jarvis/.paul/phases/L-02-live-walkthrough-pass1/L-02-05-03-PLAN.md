<objective>
## Goal
Fix content being buried behind the floating TutorialContinueButton during the tutorial walkthrough. When the Continue button is visible, increase `<main>` padding-bottom to ensure the last list items are never hidden behind it.

## Purpose
This is the last blocker for the Jenny demo checklist. Item 5 — "Tasks list visible — not buried under nav" — was observed during live walkthrough: when the tutorial is active, the TutorialContinueButton (fixed, floating above the tab bar) overlaps the bottom of the scroll area, hiding the last ~52px of content.

## Output
Modified `JarvisShell.tsx` — conditional `<main>` padding-bottom, larger when Continue button is active.
</objective>

<context>
## Source Files
@src/components/jarvis/layout/JarvisShell.tsx

## Root Cause Analysis

The TutorialContinueButton is fixed at:
```
bottom: calc(5rem + env(safe-area-inset-bottom) + 0.75rem)
```
= 80px + safe-area + 12px = **92px + safe-area from screen bottom**

Button height ≈ 40px (py-2.5 × 2 = 20px padding + text-sm ~20px)

→ Button TOP edge is at **132px + safe-area from screen bottom**

Current `<main>` pb = `calc(5rem + env(safe-area-inset-bottom))` = **80px + safe-area**

Gap: 132px - 80px = **52px of content hidden behind the button**

## Fix

When `TutorialContinueButton` is visible, increase `<main>` pb to `10rem + safe-area` (160px + safe-area), giving 28px clearance above the button's top edge.

Visibility condition (mirrors `TutorialContinueButton` logic exactly):
```tsx
const continueBtnVisible = tutorialEngine.isActive || !!suggestedNext;
```

JarvisShell already:
- Imports `useTutorialStore`
- Has `tutorialEngine` from `useTutorialEngine()`
So only one new selector is needed.
</context>
