## What Was Done This Session

### H-01 Plan Audit ("Is This Your Best Work?")

Deep-read all 10 source files the plan touches (types.ts, BriefingBuilder.ts, chatProcessor.ts, tools.ts, personalStore.ts, useJarvisFetch.ts, schemas.ts, NotionClient.ts, withRetry.ts, CircuitBreaker.ts) + errorClassifier.ts. Found **3 issues** in the plan created by a prior session and corrected them:

**Fix 1 — RFC 3339 Compliance (MODERATE):**
Google Calendar API requires RFC 3339 timestamps with mandatory timezone offset. Plan originally constructed bare datetimes like `${todayStr}T00:00:00`. Added `getTimezoneOffsetString()` utility to GoogleCalendarClient.ts, updated all time boundary constructions across Task 1 (calendarToolExecutor) and Task 2 (BriefingBuilder morning/evening/weekly) to use `${dateStr}T00:00:00${tzOffset}`.

**Fix 2 — Evening Wrap Pre-existing Bug (SIGNIFICANT):**
BriefingBuilder.ts line 582: `deriveCalendarEvents(tomorrowTasks)` puts TOMORROW's task-derived events into `calendar.today`. The plan overlaid Google today events on top without fixing this, creating a mix. Corrected plan's Task 2 step 8 to change to `deriveCalendarEvents([...completedTasks, ...pendingTasks])` (today's tasks) and rewrote merge instructions with explicit variable names.

**Fix 3 — MCP Mode Calendar Tool Visibility (SIGNIFICANT):**
In chatProcessor.ts, when MCP is active, `toolsForBrain = localOnlyTools` — which excludes calendar tools. `query_calendar` would be invisible to Claude in MCP mode. Added step 6 to Task 3: update MCP branch to `[...localOnlyTools, ...calendarTools]`. Calendar is external-but-not-Notion (MCP only replaces Notion tools).

**Also verified (NOT issues):**
- errorClassifier.ts is generic (not Notion-specific) — Google 401/403 correctly classified as non-retryable
- `configError` permanent flag is acceptable for single-user serverless
- Time sort bug (AM/PM lexicographic) is pre-existing, not introduced by plan
