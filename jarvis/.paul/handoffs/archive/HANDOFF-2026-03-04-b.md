## 9 Remaining Bugs (Full Detail)

### P0 — Fix first (tutorial crashes)

**BUG 1: "Continue →" chip crashes chat**
- Symptom: Tapping Continue → "Jarvis overwhelmed" / "Load failed" errors
- Root: `ChatActionsRow` calls `sendMessage('continue tutorial')` → real Anthropic API call → overloads
- Fix: Chip calls `engine.skipStep()` directly — no API call
- File: `src/components/jarvis/layout/ChatOverlay.tsx` — `ChatActionsRow` function (~line 604)
  ```tsx
  // CURRENT (broken):
  onClick={() => onAction('continue tutorial')}
  // FIX:
  onClick={() => engine?.skipStep()}
  ```

**BUG 2: Continue chip missing / wrong chips during tutorial**
- Symptom: Static "Today's tasks / Briefing / What's due? / Habits" chips show during tutorial
- Root: `ChatActionsRow` only shows Continue if last message matches `/continue tutorial/i` regex; on mismatch returns `null` but engine context can be null showing wrong chips
- Fix: Always show Continue chip when `engine?.isActive`, regardless of message content
- File: same as BUG 1 — replace the `if (engine?.isActive)` block:
  ```tsx
  if (engine?.isActive) {
    return (
      <div className="px-4 py-2 flex gap-2 border-t border-white/5 shrink-0">
        <button
          onClick={() => engine.skipStep()}
          className="flex items-center gap-1.5 px-4 py-1.5 text-xs bg-cyan-500/15 hover:bg-cyan-500/25 text-cyan-400 rounded-full border border-cyan-500/25 transition-colors font-medium"
        >
          Continue <ChevronRight className="w-3.5 h-3.5" />
        </button>
      </div>
    );
  }
  ```

**BUG 3: Tutorial says "go to Home" for tasks/habits/bills/calendar**
- Symptom: Claude says "go to Home" but tasks etc. are under Personal
- Root: System prompt has no navigation map. Claude doesn't know app structure.
- Fix: Add navigation section to `src/lib/jarvis/intelligence/systemPrompt.ts` after the CAPABILITIES section (~line 386):
  ```
  APP NAVIGATION:
  - Home tab (bottom bar, house icon) → Quick Actions, Academy widget, Briefing — NOT tasks/habits/bills
  - Personal domain (violet icon in top domain rail) → Tasks, Habits, Bills & Finance, Calendar, Meals & Kitchen, Journal, Health, Goals
  - Chat tab (bottom bar) → opens Jarvis chat overlay
  - Learn tab (bottom bar) → Academy curriculum
  - Settings tab (bottom bar) → app settings
  When directing the user to tasks, habits, bills, or calendar: always say "go to Personal" not "go to Home".
  ```
