## The 10 Bugs (Triaged)

### P0 — Tutorial crashes / breaks
**BUG 1: "Continue →" chip crashes chat**
- Root: chip calls `sendMessage('continue tutorial')` → real Anthropic API call → "overloaded" / "Load failed"
- Fix: chip calls `engine.skipStep()` directly — NO API call
- File: `src/components/jarvis/layout/ChatOverlay.tsx` — `ChatActionsRow` function (~line 575)

**BUG 2: Quick action chips wrong during tutorial**
- Root: `ChatActionsRow` only shows Continue chip on `/continue tutorial/i` regex. When tutorial is active but regex misses, returns `null` correctly BUT `engine` can be null (context issue), showing wrong static chips
- Fix: Always show Continue chip when `engine?.isActive` — no conditions
- File: same as above

**BUG 3: Tutorial says "go to Home" for tasks/habits/bills/calendar**
- Root: Claude's live API responses say "Home" but these live under "Personal" domain
- Fix: Add navigation map to `src/lib/jarvis/intelligence/systemPrompt.ts` — "APP NAVIGATION: Home tab = Quick Actions/Academy/Briefing. Personal domain = Tasks, Habits, Bills & Finance, Calendar, Meals & Kitchen, Journal, Health, Goals."
- File: `src/lib/jarvis/intelligence/systemPrompt.ts` — add after CAPABILITIES section (~line 386)

### P1 — Major UX
**BUG 4: Content below bottom nav (site-wide) — snaps back**
- Root: `JarvisLayout` has `safe-area-all` class → adds `padding-bottom: env(safe-area-inset-bottom)` (34px on iPhone) to outer container → JarvisShell `h-dvh` overflows that container → `overflow-hidden` clips JarvisShell's bottom 34px → content ends 34px higher than BottomTabBar allows
- Fix: Remove `safe-area-all` from JarvisLayout div + increase main pb from `4rem` to `5rem` base
- Files: `src/app/jarvis/layout.tsx` (remove `safe-area-all`), `src/components/jarvis/layout/JarvisShell.tsx` (pb `4rem` → `5rem`)

**BUG 5: Wrong voice — Polly "contin" not ElevenLabs**
- Root: ElevenLabs is failing silently in prod, Polly fallback kicks in (Polly Neural Matthew mispronounces "Continue" as "contin")
- We CAN'T see why ElevenLabs fails without Vercel logs
- Fix: In TTS route, when ElevenLabs is configured but fails, return 503 (no Polly fallback) + log exact error body. Add `X-TTS-Service` header so browser dev tools show which was used. Also try `eleven_multilingual_v2` as model (cloned voices sometimes need this vs `eleven_turbo_v2_5`)
- File: `src/app/api/jarvis/tts/route.ts`

**BUG 6: + button does nothing**
- Root: BottomTabBar `quick-add` action just `console.log`s
- Fix: Wire to `toggleChat()` — opens chat so user can say "add habit: X"
- File: `src/components/jarvis/layout/BottomTabBar.tsx` (~line 35)

**BUG 7: Chat opens at top of conversation**
- Root: Auto-scroll fires on `messages` change, not on chat open. When chat opens, messages already exist but didn't change → no scroll
- Fix: Add useEffect for `animState === 'open'` → scrollIntoView with `behavior: 'instant'`
- File: `src/components/jarvis/layout/ChatOverlay.tsx` (~line 78)

### P2 — Usability
**BUG 8: Learn tab breaks tutorial context**
- Root: Tapping Learn navigates to AcademyHub. Tutorial engine still runs but AcademyHub shows no tutorial state. User can't see progress or return easily.
- Fix: In AcademyHub, if `tutorialStore.currentLesson !== null`, show banner "Tutorial in progress — tap to return"
- File: `src/components/jarvis/academy/AcademyHub.tsx`

**BUG 9: Spotlight tap should advance tutorial (not require chat)**
- Root: SpotlightOverlay click flashes green, clears spotlight, but tutorial engine polls independently (3s dwell for ring-type). Feels disconnected.
- Fix: SpotlightOverlay click emits `tutorialActionBus.emit('spotlight-element-tapped')`. Tutorial engine listens: when `spotlight.type === 'ring'` and `spotlight-element-tapped` fires, skip the 3s dwell and advance immediately.
- Files: `src/components/jarvis/onboarding/SpotlightOverlay.tsx`, `src/lib/jarvis/hooks/useTutorialEngine.ts`

**BUG 10: Bills→Calendar loop (Claude hallucinates calendar lesson)**
- Root: When "Continue" goes through chat API, Claude generates tutorial-like text saying "Say Continue to see Calendar" — but no calendar lesson exists. Fixed by BUG 1 fix (no API calls from Continue chip).
