## The Blocking Bug (DB + Chat 500)

**Symptom:** "Brain offline — database unreachable" + HTTP 500 on chat

**Root cause identified:** Next.js running under PM2 (`cmd /c npm start`) is NOT loading `.env.local`. Direct node test confirmed DB connects fine when env vars loaded manually — the vars are correct, just not reaching the running process.

**Fix to implement:**

Replace PM2's `cmd /c npm start` with a batch script that explicitly exports vars:

Create `jarvis/scripts/start-web.bat`:
```bat
@echo off
for /f "usebackq tokens=1,* delims==" %%a in (".env.local") do (
  if not "%%a"=="" if not "%%a:~0,1%"=="#" set "%%a=%%b"
)
npm start -- -p 3001
```

Then in `ecosystem.config.js` change jarvis-web to:
```js
script: 'jarvis/scripts/start-web.bat',
interpreter: 'cmd',
interpreter_args: '/c',
```

**Alternative (simpler):** Copy all env vars directly into the PM2 `env` block in ecosystem.config.js.
