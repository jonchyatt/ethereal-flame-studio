- **6 CRITICAL fixes implemented:**
  1. `openWithMessage` now also sets `shellStore.isChatOpen` ‚Äî Executive Bridge + QuickActions Chat buttons were silently failing in production (queued messages lost)

## Key Decisions Made

- **Dual chat state was the #1 finding** ‚Äî surfaced independently by both the Shell agent and the Stores agent. Executive Bridge "Chat" action buttons have been silently failing since G-03 shipped. Now fixed.
- **Three systemic weaknesses documented (not yet fixed ‚Äî larger refactors):**
  1. Dual rendering ‚Äî mobile + desktop rendered simultaneously (Sheet, ChatOverlay, DomainRail) instead of single responsive trees
  2. Store hygiene ‚Äî derived data stored as state, full-store destructures causing cascade re-renders, dead/zombie stores
  3. Accessibility debt ‚Äî no focus indicators on most primitives, no ARIA live regions, no focus traps, no scroll lock on modals
