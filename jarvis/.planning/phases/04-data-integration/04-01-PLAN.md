Create `src/lib/jarvis/notion/schemas.ts` with TypeScript types for:

1. NotionFilter interface for building Notion API filters

   - Task (title)

   - Due Date (date)
   - Project (relation)
   - Priority (select: Low, Medium, High)
3. BillProperties interface with fields:
   - Bill (title)
   - Amount (number)
   - Due Date (date)
   - Paid (checkbox)
   - Category (select)

5. Query builder functions:
   - `buildTaskFilter(options)`: Build filter for today/this_week/overdue
   - `buildBillFilter(options)`: Build filter for due bills
6. Result formatter functions:
   - `formatTaskResults(result)`: Convert Notion API response to readable text with IDs
   - `formatBillResults(result)`: Same for bills

NOTE: Property names are case-sensitive. Use placeholder names for now - user will update after running discovery script.

    - scripts/discover-notion-databases.ts

  <action>
Create `scripts/discover-notion-databases.ts` - a one-time utility script to:

1. Import and use NotionClient
2. Call MCP `notion-search` tool to find databases by name (search for "Tasks", "Bills")
3. For each database found, call `retrieve-a-database` to get data_source_id
4. Print the property schema for each database (so user can update TASK_PROPS/BILL_PROPS)
5. Output env var format for user to copy:
   ```
   NOTION_TASKS_DATA_SOURCE_ID=xxx
   NOTION_BILLS_DATA_SOURCE_ID=xxx
   ```
6. Clean up MCP process when done

Add to package.json scripts:
```json
"discover-notion": "npx tsx scripts/discover-notion-databases.ts"
```

Move NOTION_TOKEN from .mcp.json to .env.local:
```
NOTION_TOKEN=ntn_xxx (copy from .mcp.json)
```

The script should:
- Handle errors gracefully (database not found, permission issues)
- Show clear instructions for user to share databases with integration
- Output ready-to-copy env vars
