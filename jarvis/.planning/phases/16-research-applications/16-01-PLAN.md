Follow the exact same pattern as `schedulerTools.ts`. Import `ToolDefinition` from `'../intelligence/tools'`. Export `researchTools: ToolDefinition[]` with 4 tools and `researchToolNames: Set<string>` with the 4 tool names.

The 4 tools:

1. **`store_research`** -- Store a structured research finding.
   - description: `"Store a structured research finding in the research library. Use when you've found a specific fact, deadline, eligibility criterion, amount, or other structured data point about a grant, credit option, business entity, or any topic."`
   - properties:
     - `domain` (string, required): `"Category: 'grant' | 'credit' | 'business' | 'general'"`
     - `topic` (string, required): `"Topic name, e.g., 'Amber Grant', 'Satori Living Foundation', 'Chase Ink Business Card'"`
     - `field_name` (string, required): `"Structured field name: 'deadline', 'eligibility', 'amount', 'requirements', 'application_url', 'ein', 'mission_statement', etc."`
     - `field_value` (string, required): `"The value for this field"`
     - `source` (string): `"URL or document reference where this was found"`
     - `confidence` (string): `"Confidence level: 'high' | 'medium' | 'low'. Default: 'medium'"`
     - `notes` (string): `"Additional context, caveats, or qualifications"`
     - `tags` (string): `"JSON array of tags, e.g., '[\"urgent\",\"march-2026\",\"nonprofit-only\"]'"`
     - `expires_at` (string): `"ISO date when this finding expires (e.g., grant deadline date)"`
   - required: `['domain', 'topic', 'field_name', 'field_value']`

2. **`search_research`** -- Search the research library.
   - description: `"Search the research library for findings matching a keyword query. Searches across topic, field names, field values, notes, and tags. Optionally filter by domain. Use when user asks 'what grants am I eligible for?', 'what do we know about X?', or 'find research on Y'."`
   - properties:
     - `query` (string, required): `"Search query (keyword-based, searches across all text fields)"`
     - `domain` (string): `"Optional domain filter: 'grant' | 'credit' | 'business' | 'general'"`
   - required: `['query']`

3. **`get_research_topic`** -- Get all entries for a specific domain+topic.
   - description: `"Get all research entries for a specific domain and topic combination. Returns the full profile of a grant, business entity, or credit option with all stored fields. Use when you need complete details, e.g., 'show me everything about the Amber Grant'."`
   - properties:
     - `domain` (string, required): `"Domain: 'grant' | 'credit' | 'business' | 'general'"`
     - `topic` (string, required): `"Topic name to retrieve"`
   - required: `['domain', 'topic']`

4. **`list_research_topics`** -- List all topics in the library.
   - description: `"List all unique domain and topic combinations in the research library with entry counts. Shows what research has been stored: available grants, business profiles, credit options, etc."`
   - properties:
     - `domain` (string): `"Optional domain filter"`
   - required: none (empty object is valid)

The `researchToolNames` set: `new Set(['store_research', 'search_research', 'get_research_topic', 'list_research_topics'])`

**Create `src/lib/jarvis/research/researchToolExecutor.ts`:**

Follow the exact same pattern as `scheduler/toolExecutor.ts`. Import from `'./researchStore'`. Export `executeResearchTool(name: string, input: Record<string, unknown>): Promise<string>`.

Switch on tool name:

- `store_research`: Call `saveResearchEntry()` with mapped fields (`input.field_name` -> `fieldName`, `input.field_value` -> `fieldValue`, `input.expires_at` -> `expiresAt`). Default confidence to `'medium'` if not provided. Return `JSON.stringify({ stored: true, id: entry.id, topic: entry.topic, domain: entry.domain })`.

- `search_research`: Call `searchResearch(input.query as string, input.domain as string | undefined)`. If no results, return `'No research findings match that query.'`. Otherwise return `JSON.stringify(results, null, 2)`.

- `get_research_topic`: Call `getResearchByTopic(input.domain as string, input.topic as string)`. If no results, return `` `No research found for ${input.domain}/${input.topic}.` ``. Otherwise return `JSON.stringify(entries, null, 2)`.

- `list_research_topics`: Call `listResearchTopics(input.domain as string | undefined)`. If empty, return `'Research library is empty.'`. Otherwise return `JSON.stringify(topics, null, 2)`.

- `default`: Throw `new Error(`Unknown research tool: ${name}`)`.
