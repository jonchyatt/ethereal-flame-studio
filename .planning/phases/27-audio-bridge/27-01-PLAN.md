**Step 2: Create src/lib/audio/AudioExporter.ts**

This module wraps PreAnalyzer's raw FFT data and computes all 37 features. Key implementation details:

1. **8 frequency bands** -- compute from PreAnalyzer's FFT magnitudes array. With 44.1kHz sample rate and 512 FFT size, each bin = 86.1Hz. Map bins to bands:
   - sub_bass: bins 0-0 (0-86Hz, covers 20-60Hz)
   - bass: bins 1-2 (86-258Hz, covers 60-250Hz)
   - low_mid: bins 3-5 (258-517Hz, covers 250-500Hz)
   - mid: bins 6-23 (517-1982Hz, covers 500-2kHz)
   - upper_mid: bins 24-46 (1982-3962Hz, covers 2-4kHz)
   - presence: bins 47-69 (3962-5948Hz, covers 4-6kHz)
   - brilliance: bins 70-139 (5948-11975Hz, covers 6-12kHz)
   - air: bins 140-232 (12061-19987Hz, covers 12-20kHz)

2. **Envelope followers** -- per band, exponential decay: `env = max(band_amplitude, env * release_factor)`. Attack 5ms (configurable), release configurable per band (default 150ms). Computed frame-by-frame with state carried across frames.

3. **Onset detection** -- spectral flux per band. Compare current band amplitude to previous frame. If `(current - previous) > adaptive_threshold`, flag onset. Adaptive threshold = running mean of flux * 1.5. Cooldown of 3 frames per band.

4. **Spectral centroid** -- `sum(freq_i * mag_i) / sum(mag_i)`, normalized to 0-1 range over Nyquist.

5. **Spectral flatness** -- `geometric_mean(magnitudes) / arithmetic_mean(magnitudes)`. Use log-domain for numerical stability.

6. **RMS energy** -- `sqrt(mean(samples^2))` from the raw time-domain samples for the frame.

7. **Zero-crossing rate** -- count sign changes in time-domain samples, normalize by frame length.

8. **Stereo features** -- decode left/right channels separately (don't mix to mono for this). L/R balance = `(R_rms - L_rms) / (R_rms + L_rms)`. Stereo width = correlation between L and R (1 - abs(correlation)). Mid/side = `side_energy / (mid_energy + side_energy)` where mid = (L+R)/2, side = (L-R)/2.

9. **Chromagram** -- for each frame, compute magnitude for each of 12 pitch classes by summing bins that correspond to that note across all octaves. Use `note = round(12 * log2(freq / 440)) mod 12` to map bins to pitch classes. Normalize so max = 1.

10. **BPM + beat phase** -- use autocorrelation on RMS energy across a sliding window (~4 seconds). Find peak lag between 60-200 BPM range. Beat phase = `(frame_time * bpm / 60) mod 1`.

11. **Spectral contrast** -- difference between spectral peaks and valleys per sub-band, averaged and normalized.

12. **LUFS** -- approximate via windowed RMS with K-weighting approximation (high-shelf boost). Not true ITU BS.1770, but perceptually useful. Use 400ms window.

13. **Crest factor** -- `peak / rms` for the frame's samples, normalized: `min(1, crest / 10)`.

The module must NOT modify PreAnalyzer. It wraps it: call `preAnalyzer.analyze()` for the base analysis, then re-process the raw audio buffer with the expanded 8-band + feature extraction.

```typescript
import { preAnalyzer } from './PreAnalyzer';
import type { PreAnalysisResult, AudioExportResult, ExportedAudioFrame } from '@/types/index';

// Band boundary constants (bin indices for 44.1kHz / 512 FFT)
const BAND_BOUNDARIES = { ... } as const;

// Feature metadata (self-documenting)
const BAND_DEFINITIONS: AudioBandDefinition[] = [ ... ];
const FEATURE_DESCRIPTIONS: AudioFeatureDescription[] = [ ... ];

export class AudioExporter {
  /**
   * Analyze audio file and return full 37-feature export result.
   * Decodes audio, runs 8-band FFT per frame, computes all derived features.
   */
  async analyzeForExport(
    audioFile: File,
    options?: { fps?: number; onProgress?: (pct: number) => void; signal?: AbortSignal }
  ): Promise<AudioExportResult>;

  /**
   * Download the export result as a JSON file.
   * Creates a Blob, triggers browser download.
   */
  downloadJSON(result: AudioExportResult, filename?: string): void;

  /**
   * Classify the audio and suggest a preset.
   */
  classify(result: AudioExportResult): AudioClassificationHints;
}

export const audioExporter = new AudioExporter();
```

The `analyzeForExport` method must:
1. Decode audio using AudioContext (keep stereo channels for L/R features)
2. Process frame-by-frame: extract FFT magnitudes, compute 8 bands, envelopes, onsets, spectral features, stereo, chromagram, dynamics
3. After all frames: compute BPM via autocorrelation, then fill beat_phase per frame
4. Build summary statistics
5. Run auto-classification (BPM + centroid + crest factor)
6. Return complete AudioExportResult

The `downloadJSON` method must:
1. `JSON.stringify(result, null, 2)` for readability
2. Create Blob with type `application/json`
3. Create object URL, click invisible `<a>` element, revoke URL

**Self-documenting metadata** -- BAND_DEFINITIONS must include all 8 bands with name, key, min_hz, max_hz, description, typical_use. FEATURE_DESCRIPTIONS must cover all 37 features: 8 band amplitudes, 8 envelopes, 8 onsets, 4 spectral, 3 stereo, 4 musical, 2 dynamics.

Auto-classification logic:
- BPM > 120 AND crest_factor > 0.6 => "EDM" (high energy, punchy)
- BPM < 90 AND centroid < 0.3 => "Meditation" (slow, dark)
- Otherwise => "Cinematic" (default)
