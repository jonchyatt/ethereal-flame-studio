Blender compositor depth-related nodes:
- CompositorNodeRLayers: outputs["Depth"] gives Z-pass (distance from camera per pixel)
- CompositorNodeImage: loads external image/sequence (depth map files)
- CompositorNodeMapRange: remaps depth values (0-far to 0-1 range)
- CompositorNodeMath: arithmetic on depth values (compare, multiply)
- CompositorNodeSetAlpha: set alpha channel based on depth comparison
- CompositorNodeAlphaOver: composite with adjusted alpha

Depth-aware compositing algorithm:
1. Load real footage as background (Image node or Movie Clip)
2. Load depth map for the footage (from Video Depth Anything)
3. Render virtual effects with Z-pass enabled
4. Compare Z-pass (virtual depth) vs footage depth map
5. Where virtual depth > footage depth (virtual is farther), reduce virtual alpha to 0
6. Where virtual depth < footage depth (virtual is closer), keep virtual alpha at 1
7. Alpha-over the masked virtual effects onto the real footage

**Function 1: setup_footage_background(footage_path, is_equirectangular=False)**

- Return dict with clip name, frame_count, fps, resolution, is_equirectangular

**Function 2: load_depth_map(depth_path, depth_format="sequence")**
Load an external depth map (from Video Depth Anything or similar tool) into the compositor.

Args:
  - depth_path: Path to depth map image or image sequence directory
  - depth_format: "sequence" (numbered frames like depth_0001.exr) or "single" (one image)

- Create an Image node in the compositor: `CompositorNodeImage`
- If depth_format == "sequence":
  - Load image sequence: `bpy.data.images.load(depth_path)`, set `image.source = 'SEQUENCE'`
  - Set frame range to match scene frame range
- If depth_format == "single":
  - Load single image: `bpy.data.images.load(depth_path)`, set `image.source = 'FILE'`
- Add a `CompositorNodeMapRange` after the image node to normalize depth values:
  - From Min: 0, From Max: 1 (assumes Video Depth Anything outputs 0-1 normalized depth)
  - To Min: 0, To Max: 1 (passthrough -- user can adjust)
  - Clamp: True
- Return dict with image name, format, node names
- Print JSON summary

**Function 3: composite_with_depth(footage_node_name, depth_node_name, effects_layer_name, depth_threshold=0.5, blend_width=0.02)**
The core depth-aware compositing function. Combines virtual effects onto real footage using depth comparison.

Args:
  - footage_node_name: Name of the Movie Clip node (from setup_footage_background)
  - depth_node_name: Name of the depth Image node (from load_depth_map)
  - effects_layer_name: Name of the View Layer containing virtual effects (from compositor_layers.setup_multi_layer)
  - depth_threshold: Not used directly -- depth comparison is per-pixel
  - blend_width: Softness of depth edge transition (0 = hard edge, higher = softer blend to avoid harsh cutoffs)

Implementation algorithm:
1. Get the existing compositor node tree
2. Find or create a Render Layers node for effects_layer_name
3. Enable Z-pass on that view layer: `scene.view_layers[effects_layer_name].use_pass_z = True`
4. Create the depth comparison chain:
   a. `CompositorNodeMapRange` on the effects Z-pass: map camera near..far to 0..1
   b. `CompositorNodeMath` (LESS_THAN): compare effects_depth < footage_depth
      - Input 1: normalized effects depth (from step a)
      - Input 2: footage depth map (from depth_node)
      - Output: 1.0 where effect is closer (visible), 0.0 where effect is farther (occluded)
   c. If blend_width > 0: `CompositorNodeBlur` on the comparison mask with small blur radius (blend_width * image_width) for soft edges
   d. `CompositorNodeSetAlpha` on the effects render: multiply existing alpha by the depth mask
5. `CompositorNodeAlphaOver`: footage (background) + depth-masked effects (foreground)
6. Wire final output to Composite and Viewer nodes
7. Return dict describing the compositing chain
8. Print JSON summary

**Function 4: composite_with_depth_simple(footage_path, depth_path, effects_layer_name, is_equirectangular=False)**
