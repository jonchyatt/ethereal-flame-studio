**Function 1: create_vr_camera(quality="preview")**
- Load quality preset via load_quality_preset(quality)
- Call save_before_operate(tag="vr_camera_creation")
- Create camera via get_or_create_object("efs_vr_camera", ...) -- bpy.ops.object.camera_add(location=(0, 0, 1.6)) -- 1.6m is human standing eye height
- Set camera type to PANORAMIC: `camera.data.type = 'PANO'`
- Set panorama type to equirectangular stereo: `camera.data.cycles.panorama_type = 'EQUIRECTANGULAR_STEREO'`
- Set stereo convergence distance: `camera.data.stereo.convergence_distance = 10.0` (10m -- comfortable VR viewing distance)
- Set interocular distance (IPD): `camera.data.stereo.interocular_distance = IPD_METERS`
- Set stereo convergence mode: `camera.data.stereo.convergence_mode = 'OFFAXIS'` (most natural VR 3D)
- Set scene render resolution to VR tier: `scene.render.resolution_x = preset["vr_resolution_x"]`, `scene.render.resolution_y = preset["vr_resolution_y"]`
- Enable stereo render views: `scene.render.use_multiview = True`, `scene.render.views_format = 'STEREO_3D'`
- Set stereo 3D output to top-bottom: find or create the stereo 3D format settings -- `scene.render.image_settings.stereo_3d_format.display_mode = 'TOPBOTTOM'`
- CRITICAL: Do NOT add any animation keyframes to the camera. Static position only. No rotation animation. VR camera motion causes nausea.
- Do NOT set Track To constraints (user looks around freely in VR -- constrained camera breaks VR)
- Set camera clip end to 1000m (VR environments need far clipping plane): `camera.data.clip_end = 1000`
- Set as active scene camera: `bpy.context.scene.camera = camera`
- Adjust Cycles samples: multiply base samples by vr_samples_multiplier -- `scene.cycles.samples = int(preset["cycles_samples"] * preset["vr_samples_multiplier"])`
- Print JSON summary and return dict with camera settings, resolution, stereo config

**Function 2: render_vr_stereo(output_name="vr_output", frame=None)**
- Verify active camera is panoramic: `bpy.context.scene.camera.data.type == 'PANO'` -- raise ValueError if not
- Verify multiview is enabled: `bpy.context.scene.render.use_multiview` -- raise ValueError if not
- Set file format to PNG for frame sequences (compatible with ffmpeg muxing later)
- If frame is not None, call start_single_frame_render(output_name, frame=frame)
- If frame is None, call start_render(output_name) for full animation
- Print JSON summary and return dict with output_dir, stereo mode, resolution

**Function 3: inject_vr_metadata(input_path, output_path=None)**
- If output_path is None, derive from input_path: insert "_vr" before extension
- Use subprocess.run to call ffmpeg for spatial metadata injection
- The most reliable approach for YouTube VR that avoids extra dependencies: use ffmpeg metadata tags which YouTube reads:
  ```
  ffmpeg -y -i {input_path} -c copy -metadata:s:v:0 "spherical=true" -metadata:s:v:0 "stitched=true" -metadata:s:v:0 "stereo_mode=top_bottom" {output_path}
  ```
- If ffmpeg is not found, raise FileNotFoundError with helpful message
- Verify output file exists and is larger than 0 bytes
- Print JSON summary and return dict with input_path, output_path, metadata_injected

Print JSON summary at end of each function (following fire_cinema_template pattern).
