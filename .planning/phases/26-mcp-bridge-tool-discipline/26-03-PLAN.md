Create `blender/scripts/fire_orb_poc.py` that builds a complete Mantaflow fire orb scene from an empty Blender file. The script is designed to be run in stages via execute_blender_code (scene creation, then bake, then render are separate calls).

The script must create the following scene elements:

**1. Scene setup:**
- Clear the default scene (remove default cube, light, camera)
- Set frame range: 1-30 (short test, ~1 second at 30fps)
- Set Cycles as render engine
- Set render resolution to 1920x1080

**2. Mantaflow Gas Domain (`efs_fire_domain`):**
- Create a cube scaled to (2, 2, 3) at location (0, 0, 1.5)
- Add Fluid modifier, set type to DOMAIN
- Domain type: GAS
- Resolution: 64 (prototype level -- Pitfall 7 memory wall)
- Cache type: MODULAR (data and noise baked separately)
- Cache directory: `blender/cache/fire_orb_poc/` via `set_cache_directory()`
- Vorticity: 0.5
- Dissolve smoke enabled, dissolve time: 25 frames
- Use noise: enabled, noise upres factor: 2
- Display wireframe (not solid) so it does not obscure the fire visually

**3. Flow Object (`efs_fire_flow`):**
- Create an Icosphere (subdivisions=3, radius=0.3) at location (0, 0, 0.5)
- Add Fluid modifier, set type to FLOW
- Flow type: FIRE (not SMOKE or BOTH for this POC)
- Flow behavior: INFLOW
- Fuel amount: 1.0 (this is the parameter to keyframe for audio -- Pitfall 6, keyframe Flow not Domain)
- Temperature: 2.0
- Surface emission: 1.5
- Hide the flow object from render (render_set(False))

**4. Principled Volume Material (`efs_fire_material`):**
- Create material for the Domain object
- Use Principled Volume node
- Blackbody Intensity: 1.0 (enables temperature-based coloring)
- Density: 5.0
- Color: (1.0, 0.4, 0.05, 1.0) -- warm orange
- The Blackbody shader produces temperature-based coloring automatically:
  - ~1000K = deep red (edges)
  - ~2000K = orange (body)
  - ~4000-6500K = yellow-white (core)
- This material goes on the Domain, NOT the Flow object

**5. Camera (`efs_fire_camera`):**
- Position: (5, -5, 3)
- Point at the fire orb center: (0, 0, 1.5)
- Use a Track To constraint targeting an Empty at (0, 0, 1.5)
- Focal length: 50mm

**6. Lighting:**
- Add a Point Light (`efs_fire_key_light`) at (3, -2, 5), energy: 100W
- Set world background to black (0, 0, 0) -- dark background for fire contrast

**7. Scene configuration:**
- Cycles samples: 128 (with OpenImageDenoise, sufficient for preview quality)
- Film transparent: False (black background)
- Enable Bloom via compositor if available, otherwise skip

**Implementation structure — three callable functions:**

```python
def create_fire_orb_scene():
    """Create the complete scene. Run first via execute_blender_code."""
    # All scene setup code here
    # Returns JSON summary of created objects

def bake_fire_orb():
    """Start async bake. Run second, after create_fire_orb_scene()."""
    from async_bake import start_bake
    start_bake(domain_name="efs_fire_domain", bake_type="ALL", clean_first=True)

def render_fire_orb(frame=15):
    """Start async render of a single frame. Run third, after bake completes."""
    from async_render import start_single_frame_render
    start_single_frame_render(
        output_name="fire_orb_poc",
        frame=frame,
        samples=128,
        engine="CYCLES",
    )
```

**Critical constraints (from Pitfalls research):**
- Pitfall 6: Keyframe the FLOW object (fuel_amount, temperature), NOT the Domain. The POC does not keyframe yet (static fire) but the structure is set up correctly for Phase 27 audio keyframes.
- Pitfall 4: Cache directory is set to `blender/cache/fire_orb_poc/` via `set_cache_directory()`.
- Pitfall 3: `save_before_operate()` called before scene creation.
- Pitfall 7: Resolution 64 for prototype. Do NOT increase for the POC.
- Pitfall 14: If Cycles renders produce glitched frames, the script should note this. The POC is 1 frame so individual inspection is trivial.

**Usage pattern (three separate execute_blender_code calls):**

Call 1: Create scene
```python
exec(open('.../fire_orb_poc.py').read()); create_fire_orb_scene()
```

Call 2: Start bake (returns immediately, bake runs async)
```python
exec(open('.../fire_orb_poc.py').read()); bake_fire_orb()
```

Call 3: Poll until bake complete
```python
exec(open('.../poll_status.py').read()); poll_status()
```

Call 4: Render a frame (returns immediately)
```python
exec(open('.../fire_orb_poc.py').read()); render_fire_orb(frame=15)
```

Call 5: Poll until render complete
```python
exec(open('.../poll_status.py').read()); is_render_active()
```
