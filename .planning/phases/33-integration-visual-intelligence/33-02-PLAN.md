  <action>
Create `blender/scripts/batch_render.py` with the following structure:

1. **Module docstring** explaining this is the batch render queue processor for overnight unattended rendering. Usage: `from batch_render import run_batch; run_batch("path/to/jobs.json")`.

2. **sys.path setup** using the same `__file__` fallback pattern as other EFS scripts.

3. **Constants:**
   - `JOB_QUEUE_PATH = REPO_ROOT / "blender" / "jobs" / "render_queue.json"` (default location)
   - `BATCH_LOG_PATH = REPO_ROOT / "blender" / "jobs" / "batch_log.json"` (completion log)

4. **Job queue JSON schema** (documented in docstring):
```json
{
  "jobs": [
    {
      "id": "fire-meditation-01",
      "template": "fire",
      "quality": "preview",
      "audio_json": "C:/.../audio.json",
      "preset": "fire_cinema",
      "output_name": "fire_meditation",
      "vr": false,
      "enabled": true
    }
  ]
}
```
   Fields: `id` (unique string), `template` ("fire"|"water"|"combo"|"edm"|"luminous"), `quality` (quality preset name), `audio_json` (optional path to audio JSON), `preset` (optional audio mapping preset name), `output_name` (render output prefix), `vr` (optional bool, add VR camera + render VR too), `enabled` (optional bool, skip if false). Template-specific: `hdri_path` for combo, `mask_dir` and `particle_mode` for luminous.

5. **`create_job_queue(jobs, path=None)`** function that writes a jobs list to JSON. Creates `blender/jobs/` directory if missing. Returns the path written.

6. **`_run_single_job(job)`** internal function that:
   a. Calls the correct `create_*_scene()` based on `job["template"]`
   b. If `audio_json` provided, calls the correct `apply_*_audio()` with `preset`
   c. If template needs bake (fire, combo, luminous -- NOT water or edm), calls correct `bake_*()` function
   d. **Bake wait loop**: after starting bake, polls `poll_status()` every 5 seconds until status is "complete" or "error". Uses `import time; time.sleep(5)` between polls. Timeout after 3600 seconds (1 hour) -- log error and move to next job.
   e. Calls the correct `render_*()` function
   f. **Render wait loop**: polls `is_render_active()` every 10 seconds until render completes. Timeout after 7200 seconds (2 hours).
   g. If `job.get("vr")`, also calls `create_vr_camera()` and `render_vr_stereo()`
   h. Returns dict: `{"id": job_id, "status": "complete"|"error", "error": str|None, "duration_seconds": float}`

7. **`run_batch(queue_path=None)`** main entry point:
   a. Loads job queue JSON from `queue_path` or `JOB_QUEUE_PATH`
   b. Filters to `enabled != false` jobs
   c. Prints batch summary: N jobs, estimated time
   d. Iterates jobs sequentially, calling `_run_single_job()` for each
   e. Writes results to `BATCH_LOG_PATH` after each job (append, not overwrite -- so partial progress is preserved if Blender crashes)
   f. Prints final summary: N succeeded, N failed, total duration
   g. Returns list of result dicts

8. **Error handling**: each job wrapped in try/except. Failed jobs log the error and continue to the next job. Never crash the entire batch.

9. **Import bpy at top level** (this runs inside Blender). Import template modules lazily inside `_run_single_job()`.

  <action>
Create `blender/scripts/pipeline.py` with the following structure:

1. **Module docstring** explaining this is the end-to-end pipeline: audio file in, rendered video out. Usage: `from pipeline import run_pipeline; run_pipeline("C:/.../audio.json", "fire", "fire_cinema", "preview")`.

2. **sys.path setup** using the same `__file__` fallback pattern.

3. **`SUPPORTED_TEMPLATES`** dict mapping template names to their module functions:
```python
SUPPORTED_TEMPLATES = {
    "fire": {
        "create": "fire_cinema_template.create_fire_scene",
        "audio": "fire_cinema_template.apply_audio",
        "bake": "fire_cinema_template.bake_fire",
        "render": "fire_cinema_template.render_fire",
        "default_preset": "fire_cinema",
        "default_output": "fire_pipeline",
        "needs_bake": True,
    },
    "water": { ... "needs_bake": False ... },
    "combo": { ... "needs_bake": True ... },
    "edm": { ... "needs_bake": False ... },
    "luminous": { ... "needs_bake": True ... },
}
```

4. **`_wait_for_bake(timeout=3600)`** helper that polls `poll_status()` every 5 seconds until bake completes or times out. Returns True on success, False on timeout/error. Prints progress dots.

5. **`_wait_for_render(timeout=7200)`** helper that polls `is_render_active()` every 10 seconds until render completes or times out. Returns True on success, False on timeout/error.

6. **`run_pipeline(audio_json, template, preset=None, quality="preview", output_name=None, vr=False, **kwargs)`** main orchestrator:
   a. Validates `template` is in SUPPORTED_TEMPLATES
   b. Defaults `preset` from template config's `default_preset` if not specified
   c. Defaults `output_name` from template config's `default_output` if not specified
   d. **Step 1 - Create scene**: calls the template's create function with quality (and kwargs like `hdri_path`, `mask_dir`, `particle_mode` passed through)
   e. **Step 2 - Apply audio**: calls the template's apply_audio function with audio_json and preset
   f. **Step 3 - Bake** (if `needs_bake`): calls the template's bake function, then `_wait_for_bake()`
   g. **Step 4 - Render**: calls the template's render function with output_name
   h. **Step 5 - Wait for render**: calls `_wait_for_render()`
   i. **Step 6 - VR** (if `vr=True`): calls `create_vr_camera(quality)` then `render_vr_stereo(output_name + "_vr")`
   j. Prints pipeline summary: total duration, output location
   k. Returns dict: `{"status": "complete", "template": template, "quality": quality, "preset": preset, "output_name": output_name, "duration_seconds": float, "steps_completed": ["create", "audio", "bake", "render", ...]}`

7. **`list_templates()`** helper that prints SUPPORTED_TEMPLATES as a formatted table.

8. **Timing**: use `time.time()` to track total and per-step duration. Print `[pipeline] Step N: {step_name} ({duration:.1f}s)` after each step.

9. **Error handling**: each step wrapped in try/except. On failure, print which step failed, what was already completed, and return partial result dict with `"status": "error"` and `"failed_at": step_name`.
