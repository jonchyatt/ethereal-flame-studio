Create `blender/presets/visual_principles.json` with 5 perceptual principles, each containing:
- `name`: principle name
- `description`: what it means visually (1-2 sentences)
- `source`: "Decoded from analysis of professional audio-reactive VFX (UON Visuals, Beeple, Electric Sheep). Chrome MCP live analysis deferred to interactive session."
- `parameters`: dict mapping Blender parameter paths to recommended values

**Principle 1: fractal_detail**
Multi-scale visual detail from macro turbulence down to sub-pixel noise. The human eye perceives "richness" when detail exists at 3+ scales simultaneously.
Parameters:
- `fire_domain.vorticity`: 1.2 (up from 1.0 -- more large-scale turbulence)
- `fire_domain.noise_scale`: minimum 3 even at preview quality (more fine detail)
- `fire_domain.flame_vorticity`: 2.0 (up from 1.5 -- more flame-specific micro-turbulence)
- `fire_material.density`: 10.0 (up from 8.0 -- more visible micro-structure in the volume)
- `compositor.glare_size`: 8 (up from 7 -- slightly wider glow picks up more detail in bloom)
Applies to: fire, combo, luminous

**Principle 2: emergent_complexity**
Multiple independent visual systems layered together produce patterns no single system could generate. This is the "more than the sum of parts" effect.
Parameters:
- `audio_mapping.min_simultaneous_mappings`: 8 (already enforced by AUD4-05, document why)
- `fire_domain.flame_smoke`: 0.7 (up from 0.5 -- more smoke intermixed adds an independent visual layer)
- `compositor.color_balance.slope`: (1.08, 1.0, 0.92) (stronger warm/cool split adds color complexity)
- `lighting.rim_energy`: 70 (up from 50 -- stronger rim creates a third visual layer against key and self-illumination)
Applies to: fire, combo, edm, luminous

**Principle 3: contrast_darkness**
The most striking visuals emerge from darkness. Bright effects against pure black maximize perceptual impact. Professional VFX uses 70%+ frame darkness.
Parameters:
- `world.background_strength`: 0.0 (pure black -- already used in EDM, should be default for all)
- `lighting.key_energy`: 100 (DOWN from 150 for fire -- let the fire be primary illumination, not the lights)
- `lighting.ground_bounce_energy`: 15 (DOWN from 30 -- less ambient fill = deeper blacks)
- `compositor.color_balance.offset`: (0.98, 0.93, 0.85) (slightly darker shadows than current 1.0/0.95/0.88)
- `compositor.bloom_threshold`: 0.7 (DOWN from 0.8 -- more of the fire participates in bloom, but not as aggressive as EDM's 0.5)
Applies to: fire, water, combo, luminous

**Principle 4: sync_precision**
Audio-visual synchronization must be frame-accurate. Latency of even 2 frames (66ms at 30fps) is perceived as "off". The sync itself should be non-linear: onset events hit hard, envelopes decay smoothly.
Parameters:
- `keyframe.curve_type_onsets`: "CONSTANT" (instant snap for beat hits)
- `keyframe.curve_type_envelopes`: "BEZIER" with AUTO_CLAMPED handles (smooth decay)
- `audio_mapping.onset_scale`: 1.5 (amplify onset events above envelope baseline)
- `audio_mapping.envelope_smoothing`: "none" (do NOT smooth -- the audio analysis already provides per-frame values)
Applies to: all templates

**Principle 5: expectation_violation**
The most memorable moments in audio-reactive VFX violate the viewer's expectations: sudden darkness before a drop, color temperature inversion, scale flip. The dynamic_range system already handles breakdowns. This principle extends to visual surprise.
Parameters:
- `dynamic_range.breakdown_multiplier`: 0.08 (DOWN from 0.1 -- even darker breakdowns for more contrast with drops)
- `dynamic_range.drop_multiplier`: 1.2 (UP from 1.0 -- drops should momentarily exceed normal brightness)
- `color_temp.inversion_on_drop`: true (when bass onset after breakdown, briefly invert color temperature warm->cool for 3 frames)
- `camera.fov_shift_on_drop`: 0.02 (subtle shift_x on bass drops for visual "punch" -- 2% of sensor width)
Applies to: fire, combo, edm

Each principle also includes a `before_after` field describing what changes visually (for human verification): e.g., "Before: smooth uniform fire column. After: turbulent multi-scale fire with visible micro-detail at edges."

Create `blender/scripts/apply_visual_principles.py` with the following structure:

1. **Module docstring** explaining this applies visual intelligence principles to the active Blender scene. Usage: `from apply_visual_principles import apply_principles; apply_principles(["fractal_detail", "contrast_darkness"])` or `apply_principles("all")`.

2. **sys.path setup** using the standard `__file__` fallback pattern.

3. **`_load_principles()`** internal function that loads `visual_principles.json` from PRESETS_DIR. Returns the parsed dict. Caches in module-level variable after first load.

4. **`list_principles()`** function that prints all 5 principles with their names, descriptions, and which templates they apply to. Returns the list of principle dicts.

5. **`_apply_fractal_detail(scene)`** -- modifies the fire domain (if present) to increase vorticity, noise_scale, flame_vorticity. Modifies the fire material to increase density. Modifies compositor glare size. Only touches objects that exist (safe to call on any scene). Prints each parameter change.

6. **`_apply_emergent_complexity(scene)`** -- increases flame_smoke on fire domain, adjusts compositor color balance slope, increases rim light energy. Safe for any scene.

7. **`_apply_contrast_darkness(scene)`** -- sets world background strength to 0.0, reduces key light and ground bounce energy, adjusts color balance offset darker, lowers bloom threshold.

8. **`_apply_sync_precision(scene)`** -- this principle affects keyframe generation, not scene parameters. Print a message documenting the recommended keyframe settings (curve types, onset scaling). No scene modifications needed -- the keyframe_generator already supports these settings via preset `curve_type` fields.

9. **`_apply_expectation_violation(scene)`** -- adjusts dynamic_range breakdown/drop multipliers. Print a message documenting the recommended preset changes (these are in the audio mapping preset JSON, not scene parameters). Modify bloom threshold if compositor exists.

10. **`apply_principles(principle_names, scene=None)`** main entry point:
    a. If `principle_names == "all"` or `principle_names == ["all"]`, apply all 5
    b. Otherwise, apply only the named principles
    c. Calls `save_before_operate("visual_principles")` first
    d. Dispatches to the correct `_apply_*()` function for each
    e. Prints summary: "Applied N principles: [names]"
    f. Returns dict with applied principles and parameter changes

11. **Detection logic**: each `_apply_*` function checks for the existence of relevant objects before modifying. Uses `bpy.data.objects.get("efs_fire_domain")`, etc. If object not found, prints skip message and continues.

12. **Compositor access pattern**: `scene.node_tree.nodes.get("Glare")`, `scene.node_tree.nodes.get("Color Balance")` -- same pattern as `setup_compositor()` in fire_cinema_template.py.
