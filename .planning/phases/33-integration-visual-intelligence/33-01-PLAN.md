  <action>
Create `blender/scripts/efs_cli.py` with the following structure:

1. **Module docstring** explaining this is the unified CLI harness for all EFS Blender workflows, with usage examples showing `from efs_cli import run; run(["create-fire", "preview"])`.

2. **sys.path setup** using the same `__file__` fallback pattern as fire_cinema_template.py (try `os.path.dirname(os.path.abspath(__file__))`, except NameError fall back to hardcoded scripts dir).

3. **COMMANDS dict** mapping command names to handler functions. 10 commands total:

   - `"create-fire"` -> args: `[quality]` (default "preview") -> calls `fire_cinema_template.create_fire_scene(quality)`
   - `"create-water"` -> args: `[quality]` (default "preview") -> calls `water_template.create_water_scene(quality)`
   - `"create-combo"` -> args: `[quality, hdri_path?]` (default "preview", None) -> calls `combo_fire_water.create_fire_water_scene(quality, hdri_path)`
   - `"create-edm"` -> args: `[quality]` (default "preview") -> calls `edm_light_template.create_edm_scene(quality)`
   - `"create-luminous"` -> args: `[mask_dir, quality, particle_mode?]` (required, default "preview", "flame") -> calls `luminous_being_template.create_luminous_scene(mask_dir, quality, particle_mode)`
   - `"bake"` -> args: `[type?]` (default "all") -> dispatches to the correct bake function based on what scene objects exist: checks for `efs_fire_domain` -> `bake_fire()`, `efs_combo_fire_domain` -> `bake_fire_water()`, `efs_lumi_fire_domain` -> `bake_luminous()`. If type is "ocean" -> `bake_ocean()`. Print error if no bakeable domain found.
   - `"render"` -> args: `[output_name?, frame?]` -> auto-detects scene type from object names and calls correct render function. Detection: `efs_fire_domain` without `efs_ocean_plane` -> `render_fire(output_name, frame)`, `efs_ocean_plane` without `efs_fire_domain` -> `render_water(output_name, frame)`, both -> `render_fire_water(output_name, frame)`, `efs_edm_laser_*` -> `render_edm(output_name, frame)`, `efs_lumi_body` -> `render_luminous(output_name, frame)`. Default output_name derived from detected scene type.
   - `"render-vr"` -> args: `[output_name?, frame?]` -> calls `vr_template.render_vr_stereo(output_name or "vr_output", frame)`
   - `"apply-audio"` -> args: `[audio_json_path, preset_name?]` -> auto-detects preset from scene type if not specified, calls `keyframe_generator.apply_audio_keyframes(audio_json_path, preset_name)`. Default presets: fire scene -> "fire_cinema", water -> "water_ocean", combo -> "fire_water_combo", edm -> "edm_lights", luminous -> "luminous_being".
   - `"list-presets"` -> no args -> calls `keyframe_generator.list_presets()`, prints formatted table.

4. **Lazy imports** -- template modules are imported inside each handler function (not at module top level) to avoid import errors when running in contexts where only some templates are needed. Same pattern as fire_cinema_template's `from async_bake import start_bake` inside `bake_fire()`.

5. **`_detect_scene_type()` helper** that checks `bpy.data.objects` for known EFS object name prefixes and returns one of: "fire", "water", "combo", "edm", "luminous", "unknown". Object detection rules:
   - `efs_lumi_body` present -> "luminous"
   - `efs_edm_laser_red` present -> "edm"
   - `efs_fire_domain` AND `efs_ocean_plane` present -> "combo"
   - `efs_fire_domain` present (without ocean) -> "fire"
   - `efs_ocean_plane` present (without fire) -> "water"
   - else -> "unknown"

6. **`run(args)` function** that takes a list of strings (like sys.argv), extracts the command name from args[0], looks up the COMMANDS dict, parses remaining args positionally, and calls the handler. Returns the handler's return value (dict or None). Prints `[efs_cli] Running: {command} {args}` at start and `[efs_cli] Done: {command}` on success. On unknown command, prints available commands and returns None. On error, prints traceback and returns `{"error": str(e)}`.

7. **`help()` function** that prints all commands with their argument signatures and descriptions in a formatted table.

8. **No argparse** -- keep it simple. Arguments are positional strings. This is called from `exec(open(...).read())` inside Blender, not from a terminal. The `run()` function IS the interface.
