Rewire 3 legacy poll/cancel API routes from the deprecated in-memory `audioPrepJobs` (JobManager) to the `getJobStore()` factory so that job state queries hit the same persistent store used by the worker and the newer API routes. Output: Three route files updated to use `getJobStore()` with the existing `{ success, data }` response wrapper preserved for GET handlers.
</objective>

<task type="auto">
  <name>Task 1: Rewire ingest poll/cancel route to getJobStore()</name>
  <files>src/app/api/audio/ingest/[jobId]/route.ts</files>
  <action>
Replace the entire contents of `src/app/api/audio/ingest/[jobId]/route.ts`:

1. Remove the `import { audioPrepJobs } from '@/lib/audio-prep/JobManager'` import.
2. Add `import { getJobStore } from '@/lib/jobs'` instead.
3. Rewrite the `GET` handler:
   - Call `const store = getJobStore()` then `const job = await store.get(jobId)`.
   - If `!job`, return `NextResponse.json({ success: false, error: { code: 'NOT_FOUND', message: \`Job ${jobId} not found\` } }, { status: 404 })`.
   - On success, return `NextResponse.json({ success: true, data: { jobId: job.jobId, type: job.type, status: job.status, progress: job.progress, stage: job.stage, result: job.result, error: job.error, createdAt: job.createdAt, updatedAt: job.updatedAt } })`.
   - Wrap in try/catch returning 500 on error.
4. Rewrite the `DELETE` handler:
   - Call `const store = getJobStore()` then `const job = await store.get(jobId)`.
   - If `!job`, return 404 with `{ error: 'Job not found' }`.
   - If job is in a terminal state (`complete`, `failed`, `cancelled`), return 409 with `{ error: 'Job already in terminal state', status: job.status }`.
   - Call `await store.cancel(jobId)`.
   - Return `NextResponse.json({ jobId, status: 'cancelled', message: 'Job cancellation requested' })`.
   - Wrap in try/catch returning 500 on error.

**CRITICAL: The GET handler MUST preserve the `{ success: true/false, data: {...} }` response wrapper.** AudioPrepEditor.tsx polling loops parse `data.success` and `data.data.*` fields directly. Dropping this wrapper will break all polling. The DELETE handler can use the flat shape (AudioPrepEditor does not parse DELETE responses).

Keep the existing HTTP semantics: 404 for missing, 200 for found, 409 for terminal cancel attempt.
