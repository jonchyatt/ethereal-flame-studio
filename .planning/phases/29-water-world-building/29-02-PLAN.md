**Functions:**

1. `setup_hdri(hdri_path, strength=1.0, rotation_z=0.0)`:
   - Args: `hdri_path` (absolute path to .exr/.hdr file), `strength` (environment light intensity, default 1.0), `rotation_z` (rotate HDRI in radians, default 0.0 -- useful for controlling sun direction)
   - Get or create World: `bpy.data.worlds.get("World")` or `bpy.data.worlds.new("World")`, assign to scene
   - Enable nodes: `world.use_nodes = True`
   - Clear existing world node tree: `world.node_tree.nodes.clear()`
   - Create nodes:
     - Environment Texture node: `nodes.new(type='ShaderNodeTexEnvironment')`, set location (-600, 0). Load image: `node.image = bpy.data.images.load(str(hdri_path))` (absolute path). If file not found, raise FileNotFoundError with helpful message mentioning polyhaven_download MCP tool.
     - Mapping node: `nodes.new(type='ShaderNodeMapping')`, set location (-900, 0). Set `mapping.inputs["Rotation"].default_value[2] = rotation_z` (Z rotation to spin the HDRI)
     - Texture Coordinate node: `nodes.new(type='ShaderNodeTexCoord')`, set location (-1200, 0)
     - Background node: `nodes.new(type='ShaderNodeBackground')`, set location (-300, 0). Set `bg.inputs["Strength"].default_value = strength`
     - World Output node: `nodes.new(type='ShaderNodeOutputWorld')`, set location (0, 0)
   - Wire: Tex Coord (Generated) -> Mapping (Vector) -> Environment Texture (Vector), Environment Texture (Color) -> Background (Color), Background (Background) -> World Output (Surface)
   - Print: `[world_template] HDRI loaded: {hdri_path}, strength={strength}, rotation_z={rotation_z}`
   - Return dict: `{"status": "hdri_loaded", "path": str(hdri_path), "strength": strength, "rotation_z": rotation_z}`

2. `place_asset(object_name, location=(0, 0, 0), rotation=(0, 0, 0), scale=(1, 1, 1))`:
   - Finds an already-imported object by name and positions it. Assets are imported via blender-mcp MCP tools (search_sketchfab + import_sketchfab_model, or hyper3d_generate + import_generated_model), which add objects to the scene. This function then positions them.
   - Look up object: `obj = bpy.data.objects.get(object_name)`
   - If not found, print list of all object names and raise ValueError: `f"Object '{object_name}' not found. Available: {[o.name for o in bpy.data.objects]}"`
   - Set location: `obj.location = location`
   - Set rotation (euler radians): `obj.rotation_euler = rotation`
   - Set scale: `obj.scale = scale`
   - Ensure it casts shadows: no special config needed (Cycles default)
   - Print: `[world_template] Asset placed: {object_name} at {location}, rot={rotation}, scale={scale}`
   - Return dict: `{"status": "asset_placed", "name": object_name, "location": list(location), "rotation": list(rotation), "scale": list(scale)}`

3. `create_ground_plane(size=50, material_color=(0.15, 0.12, 0.1, 1.0))`:
   - Creates a simple ground plane for scenes that need a surface under the fire/assets (but NOT water scenes, which use the ocean as ground).
   - `get_or_create_object("efs_world_ground", lambda: bpy.ops.mesh.primitive_plane_add(size=size, location=(0, 0, -0.01)))` -- slightly below origin
   - Create a simple diffuse material: `"efs_ground_material"`, Principled BSDF with `Base Color = material_color`, `Roughness = 0.9` (matte ground)
   - Assign material
   - Smooth shading
   - Return ground object

4. `setup_world(hdri_path=None, hdri_strength=1.0, hdri_rotation_z=0.0, ground=False, ground_size=50, ground_color=(0.15, 0.12, 0.1, 1.0))`:
   - Convenience wrapper that sets up a complete world environment in one call.
   - If `hdri_path` is provided, call `setup_hdri(hdri_path, hdri_strength, hdri_rotation_z)`
   - If `ground` is True, call `create_ground_plane(ground_size, ground_color)`
   - Print summary
   - Return dict with combined results

5. `list_scene_objects()`:
   - Utility that prints all objects currently in the scene with their names, types, and locations. Helpful for finding the name of an imported Sketchfab/AI asset to pass to `place_asset()`.
   - Iterate `bpy.data.objects`, print each: `[world_template] {name} ({type}) at {location}`
   - Return list of dicts

**Important design notes to include in docstring:**
- HDRIs must be downloaded FIRST via blender-mcp MCP tools (polyhaven_search + polyhaven_download). This script only loads the file into the world node tree.
- 3D assets must be imported FIRST via blender-mcp MCP tools (search_sketchfab + import_sketchfab_model, or hyper3d_generate + import_generated_model). This script only positions them.
- This two-step design keeps Python scripts independent of the MCP transport layer.
