  <action>
Create `blender/scripts/water_template.py` following the exact same module pattern as `fire_cinema_template.py`. The script provides a four-function API: `create_water_scene()` -> `apply_water_audio()` -> `bake_ocean()` -> `render_water()`.

**Module header:** Same docstring pattern as fire_cinema_template.py -- describe what it does, the four-function API with MCP execute_blender_code usage examples (6 calls: create, apply audio, bake, poll, render, poll), and pitfall compliance notes.

**sys.path + imports:** Same `__file__` fallback pattern. Import `save_before_operate`, `get_or_create_object`, `RENDERS_DIR` from scene_utils. Import `load_quality_preset` from fire_cinema_template (reuse the same quality presets). Import `bpy`, `json`, `math`, `sys`, `os`, `Path`.

**Internal functions (all prefixed with underscore):**

1. `_clear_default_scene()` -- Same as fire_cinema_template: `select_all` + `delete`.

2. `_setup_scene(preset)` -- Configure Cycles render settings from quality preset. Same as fire_cinema_template._setup_scene() EXCEPT:
   - `scene.render.film_transparent = False` (opaque for water reflections)
   - Enable caustics: `scene.cycles.caustics_reflective = True`, `scene.cycles.caustics_refractive = True`
   - Set `scene.cycles.max_bounces = 12`, `scene.cycles.transparent_max_bounces = 8`, `scene.cycles.glossy_bounces = 4`, `scene.cycles.transmission_bounces = 12` (needed for water refraction + caustics)
   - World background: dark blue-black `(0.001, 0.002, 0.005, 1.0)` with strength 1.0

3. `_create_ocean(preset)` -- Create an ocean surface:
   - Use `get_or_create_object("efs_water_ocean", lambda: bpy.ops.mesh.primitive_plane_add(size=40, location=(0, 0, 0)))` -- large 40x40 plane at origin
   - Add Ocean Modifier: `ocean.modifiers.new(name="Ocean", type='OCEAN')`
   - Configure ocean modifier settings:
     - `mod.geometry_mode = 'GENERATE'`
     - `mod.repeat_x = 2`, `mod.repeat_y = 2` (tileable)
     - `mod.resolution = 12` (for draft/preview) or `16` (production/ultra) -- use `16 if preset["resolution_max"] >= 256 else 12`
     - `mod.spatial_size = 25` (meters, controls wave wavelength -- larger = longer waves)
     - `mod.depth = 200` (deep ocean behavior)
     - `mod.size = 1.0` (base wave height -- this is the keyframeable parameter for audio)
     - `mod.choppiness = 1.5` (base choppiness -- also keyframeable for audio)
     - `mod.scale = 1.0`
     - `mod.use_foam = True` (enable foam data output)
     - `mod.foam_coverage = 0.3` (foam only on wave crests)
     - `mod.foam_layer_name = "foam"` (vertex color layer name)
     - `mod.time = 0` (will be animated)
   - Animate ocean time: Insert keyframe at frame 1 with `mod.time = 0` and at the last frame (`preset["frame_count"]`) with `mod.time = preset["frame_count"] / 24.0`. Use `ocean.keyframe_insert(data_path='modifiers["Ocean"].time', frame=1)` and same at last frame. This makes the ocean surface animate smoothly.
   - Smooth shading: `bpy.ops.object.shade_smooth()` with ocean selected
   - Return the ocean object

4. `_create_water_material(ocean)` -- Create a physically-based water material:
   - Material name: `"efs_water_material"`
   - Use nodes. Clear existing nodes.
   - Create Glass BSDF node: `nodes.new(type='ShaderNodeBsdfGlass')`
     - `glass.inputs["Color"].default_value = (0.8, 0.9, 1.0, 1.0)` (slightly blue tint)
     - `glass.inputs["Roughness"].default_value = 0.05` (nearly mirror-smooth)
     - `glass.inputs["IOR"].default_value = 1.333` (water IOR)
   - Create Material Output node, connect Glass BSDF -> Surface output
   - Assign material to ocean object
   - Return the material

5. `_create_foam_particles(ocean)` -- Create spray/foam particle system on the ocean:
   - Add particle system: `ocean.modifiers.new(name="FoamParticles", type='PARTICLE_SYSTEM')`
   - Get particle settings: `ps = ocean.particle_systems[-1].settings`
   - Configure: `ps.count = 5000`, `ps.frame_start = 1`, `ps.frame_end = preset["frame_count"]`, `ps.lifetime = 15`
   - Physics: `ps.physics_type = 'NEWTON'`, `ps.mass = 0.01`
   - Emission: `ps.emit_from = 'FACE'`, `ps.use_modifier_stack = True` (emit from deformed ocean surface)
   - Render: `ps.render_type = 'HALO'`, `ps.particle_size = 0.02` (tiny spray droplets)
   - Velocity: `ps.normal_factor = 0.5`, `ps.factor_random = 0.3` (upward spray with randomness)
   - Use vertex group for foam emission: `ps.vertex_group_density = "foam"` (from Ocean Modifier foam output)
   - Gravity influence: `ps.effector_weights.gravity = 0.5` (partially floaty)
   - Return the particle system settings object

6. `_create_camera()` -- Camera for water scene:
   - Track target empty: `get_or_create_object("efs_water_target", lambda: bpy.ops.object.empty_add(type='PLAIN_AXES', location=(0, 0, 0)))` at `(0, 0, 0)` (water surface)
   - Camera: `get_or_create_object("efs_water_camera", lambda: bpy.ops.object.camera_add(location=(8, -8, 4)))` at `(8, -8, 4)` -- elevated angle looking down at water
   - Focal length: `camera.data.lens = 35` (wider than fire to capture water expanse)
   - DOF: `camera.data.dof.use_dof = True`, `focus_distance = 11.3`, `aperture_fstop = 4.0` (slightly deeper DOF than fire)
   - Track To constraint pointing at target empty (same pattern as fire_cinema_template)
   - Set as active camera
   - Return camera object

7. `_create_lighting()` -- Lighting for water scene:
   - Key sun light: `get_or_create_object("efs_water_sun", lambda: bpy.ops.object.light_add(type='SUN', location=(0, 0, 10)))`. Energy 3.0, warm white color `(1.0, 0.98, 0.95)`, angle 15 degrees below horizon for dramatic reflections: `sun.rotation_euler = (math.radians(60), 0, math.radians(30))`
   - Area fill light: `get_or_create_object("efs_water_fill", lambda: bpy.ops.object.light_add(type='AREA', location=(-5, 5, 3)))`. Energy 20, cool blue `(0.7, 0.85, 1.0)`, size 5.0
   - Return (sun, fill)

8. `setup_water_compositor()` -- Compositor for water:
   - Same enable-compositing pattern as fire_cinema_template
   - Render Layers node
   - Glare node: `FOG_GLOW`, quality `HIGH`, mix 0.0, threshold 0.9 (higher than fire -- only specular highlights bloom), size 6
   - Color Balance node: `OFFSET_POWER_SLOPE`, offset `(0.95, 0.98, 1.0)` (cool shadows), power `(1.0, 1.0, 1.0)`, slope `(0.95, 1.0, 1.05)` (cooler highlights than fire)
   - Composite + Viewer outputs
   - Wire: Render Layers -> Glare -> Color Balance -> Composite + Viewer
   - Return dict describing setup

**Public API functions:**

1. `create_water_scene(quality="preview")` -- Main entry point:
   - Load quality preset via `load_quality_preset(quality)` (imported from fire_cinema_template)
   - `save_before_operate(tag="water_scene_creation")`
   - Call all internal functions in order: clear scene, setup scene, create ocean, create material, create foam particles, create camera, create lighting, setup compositor
   - Build and print JSON summary (same pattern as fire_cinema_template.create_fire_scene)
   - Return the summary dict

2. `apply_water_audio(audio_json_path, preset_name="water_ocean")` -- Convenience wrapper:
   - `from keyframe_generator import apply_audio_keyframes`
   - `return apply_audio_keyframes(audio_json_path, preset_name=preset_name)`

3. `bake_ocean()` -- No-op placeholder that prints a message:
   - Ocean Modifier does NOT require baking (it is procedural). Print `"[water_template] Ocean Modifier is procedural -- no bake needed. Ready to render."` and return `{"status": "no_bake_needed", "reason": "Ocean Modifier is procedural"}`.
   - NOTE: This exists only for API symmetry with fire_cinema_template. The foam particles also do not need explicit baking.

4. `render_water(output_name="water_ocean", frame=None)` -- Same pattern as fire_cinema_template.render_fire:
   - Single frame: `from async_render import start_single_frame_render; start_single_frame_render(output_name, frame)`
   - Full animation: `from async_render import start_render; start_render(output_name)`

**Naming convention:** All created objects use `efs_water_` prefix (e.g., efs_water_ocean, efs_water_camera, efs_water_sun, efs_water_fill, efs_water_target, efs_water_material).

**Print logging:** Every function prints `[water_template] ...` progress messages, same pattern as fire_cinema_template.
