<objective>
Build the person segmentation and mask-to-mesh pipeline that transforms video of a person into a Blender mesh proxy. This is the foundational prerequisite -- every other Luminous Being effect (volumetric fill, particles, fire wisps, corona) needs this body mesh to emit from.

Purpose: Without clean segmentation masks and a mesh proxy, none of the visual effects can attach to the body silhouette. This plan creates the bridge from real video to Blender geometry.

Output: Two Python scripts -- sam_segmenter.py (video to mask PNGs) and mask_to_mesh.py (mask PNGs to Blender mesh proxy with per-frame shape keys or image-driven geometry).
</objective>

Create blender/scripts/sam_segmenter.py -- a standalone Python script (does NOT import bpy -- runs outside Blender as a preprocessing step) that segments a person from video frames using SAM 2.1.

Two public functions:

1. segment_video(video_path, output_dir=None, model_type="vit_b", point_prompt=None, frame_step=1):
   - Extracts frames from video using cv2.VideoCapture
   - Loads SAM 2.1 model (facebook/sam2.1-hiera-base-plus via torch hub or segment_anything_2 pip package)
   - If point_prompt is None, uses automatic mask generation with largest-area heuristic (largest connected component = person)
   - If point_prompt is (x, y) tuple, uses that as positive point prompt for the person
   - Propagates mask across frames using SAM 2.1's video predictor for temporal consistency
   - Writes binary mask PNGs (white=person, black=background) to output_dir (default: MASKS_DIR / video_stem)
   - Returns dict: {"status": "segmented", "mask_dir": str, "frame_count": int, "resolution": [w, h]}

2. segment_frames(frame_dir, output_dir=None, model_type="vit_b", point_prompt=None):
   - Same as above but takes a directory of frame images instead of a video file
   - Useful when frames are already extracted

Implementation notes:
- Use torch + segment-anything-2 (the SAM 2.1 package from Meta)
- Model checkpoint auto-downloads on first use
- Temporal consistency via SAM 2.1's video propagation (not frame-by-frame independent segmentation)
- Mask smoothing: apply cv2.morphologyEx CLOSE operation (5x5 kernel) to remove small holes
- Mask naming: frame_0001.png, frame_0002.png, ... (4-digit zero-padded, matches Blender image sequence convention)
- Include a __main__ block for CLI usage: python sam_segmenter.py --video path/to/video.mp4 --output path/to/masks/
- Print JSON progress to stdout every 10% of frames for MCP feedback
- Do NOT import bpy -- this script runs in system Python with torch, not inside Blender

Error handling:
- If torch/SAM not installed, raise ImportError with pip install instructions
- If video cannot be opened, raise FileNotFoundError with clear message
- If no person detected in first frame, raise ValueError suggesting point_prompt usage

Create blender/scripts/mask_to_mesh.py -- a Blender Python script (imports bpy) that converts mask PNG image sequences into a mesh proxy suitable as a particle emitter, volumetric fill target, and Mantaflow flow source.

Two public functions:

1. load_masks_as_mesh(mask_dir, name="efs_lumi_body", max_vertices=2000, smooth_factor=2.0):
   - Loads first mask frame to determine body silhouette contour
   - Uses cv2.findContours on the binary mask to extract the outline
   - Simplifies contour with cv2.approxPolyDP (epsilon = 0.5% of perimeter) for manageable vertex count
   - Creates a flat 2D mesh (extruded slightly for volume -- depth = 0.1 units) in the XZ plane (standing person convention)
   - Scales mesh to match a reference height of 1.8 Blender units (average human height)
   - Centers mesh at origin
   - For animation: creates shape keys from subsequent frames. If frame count > 100, samples every Nth frame to keep shape key count under 100 (linear interpolation between keys gives smooth motion)
   - Sets up shape key drivers: each shape key has a driver linked to frame number so the mesh deforms automatically during animation playback
   - Returns the bpy.types.Object

2. create_body_proxy(mask_dir, quality="preview"):
   - High-level convenience function calling load_masks_as_mesh with quality-appropriate settings
   - Quality "draft": max_vertices=500, sample every 5th frame
   - Quality "preview": max_vertices=1000, sample every 3rd frame
   - Quality "production": max_vertices=2000, sample every frame
   - Applies Subdivision Surface modifier (level 1) for smoother silhouette
   - Applies Solidify modifier (thickness=0.15) to give the flat mesh some depth for volumetric material
   - Sets up the mesh for use as:
     a. Particle emitter: adds particle system slot (empty, configured by luminous_being_template.py later)
     b. Mantaflow flow source: compatible with Flow modifier attachment
     c. Volumetric fill: material slot ready for Principled Volume assignment
   - Names object "efs_lumi_body" using get_or_create_object for idempotency
   - Hides from render (hide_render=True) -- the raw mesh is not visible, only the effects on it
   - Returns dict: {"status": "body_proxy_created", "name": str, "vertex_count": int, "shape_keys": int, "frame_range": [start, end]}

Implementation notes:
- Import cv2 at function level (not top-level) with helpful ImportError message since Blender's Python may not have opencv
- Alternative path: if cv2 not available, fall back to bpy.ops.image.open + threshold approach using Blender's built-in image processing (less accurate but no external dependency)
- Use scene_utils.get_or_create_object for idempotent creation
- Use scene_utils.save_before_operate before mesh creation
- Print JSON status to stdout following established pattern
- efs_lumi_body naming convention consistent with efs_fire_*, efs_edm_* from prior phases

The cv2 fallback approach for environments without opencv:
- Load mask image via bpy.data.images.load()
- Read pixel data, threshold at 0.5 to get binary mask
- Simple edge detection by comparing adjacent pixels
- Creates a cruder but functional silhouette mesh
