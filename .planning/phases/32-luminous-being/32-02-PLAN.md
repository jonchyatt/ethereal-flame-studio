INTERNAL FUNCTIONS (prefix with _):

_setup_scene(preset):
- Cycles engine, resolution from preset, samples, denoiser, motion blur
- film_transparent = True (needed for compositor alpha-over between layers)
- World background: Strength=0.0 (total darkness -- the being IS the light source)
- Frame range from preset (overridden by audio when applied)

_create_body(mask_dir, quality):
- Calls create_body_proxy(mask_dir, quality) from mask_to_mesh.py
- Returns the efs_lumi_body object
- This mesh is the source for all effects but is itself hidden from render

_create_volumetric_fill(body_obj):
- Creates efs_lumi_fill: a volume cube surrounding the body mesh
- Size: slightly larger than body bounding box (1.1x scale factor on each axis)
- Material: Principled Volume with:
  - Color: warm white (1.0, 0.95, 0.85, 1.0)
  - Density: 3.0 (moderate baseline -- audio will modulate this from 1.0 to 8.0)
  - Emission Strength: 5.0 (self-illuminating glow)
  - Emission Color: warm cyan-white (0.85, 0.95, 1.0, 1.0) -- ethereal tone
  - Blackbody Intensity: 0.8 (subtle temperature coloring)
- Material name: efs_lumi_fill_material
- Keyframeable targets: Density (bass), Emission Strength (rms_energy)
- Returns the fill object

_create_particle_system(body_obj, mode="flame"):
- Creates particle system on body_obj (the mesh IS the emitter)
- Three modes matching Three.js orb:
  a. "flame" (default): Hair particles, count=2000 (preview), lifetime=30 frames, velocity normal=0.3 upward, gravity=-0.2 (upward drift), size=0.02, random size 0.5
  b. "mist": Hair particles, count=3000, lifetime=50, velocity normal=0.1 random=0.5 (soft dispersion), gravity=0.0 (floating), size=0.03, random size 0.8
  c. "solar_breath": Hair particles, count=1500, lifetime=20, velocity normal=1.0 (radial pulse outward from surface), gravity=0.0, size=0.025, physics NEWTON with 0.5 damping for deceleration
- Particle material: Emission shader (Strength=2.0, Color matching mode):
  - flame: orange-gold (1.0, 0.6, 0.1)
  - mist: cool white-blue (0.8, 0.9, 1.0)
  - solar_breath: warm white (1.0, 0.95, 0.9)
- Material name: efs_lumi_particle_mat_{mode}
- Creates a small icosphere (radius=0.01) as render instance object for particle display
- Keyframeable: particle count (via scale on emission), emission strength (mid frequency)
- Returns the particle system settings object

_create_fire_wisps(body_obj, preset):
- Creates a SMALLER Mantaflow fire domain surrounding the body (efs_lumi_fire_domain)
- Domain size: body bounding box + 0.5 margin on each side
- Domain location: centered on body
- Resolution: preset resolution_max * 0.5 (lower than standalone fire -- wisps are accent, not hero)
- Mantaflow settings tuned for SMALL wisps (not towering flames):
  - vorticity: 0.8 (less than standalone fire 1.0)
  - dissolve_speed: 8 (faster dissolve = shorter wisp trails)
  - flame_max_temp: 2.0 (lower = subtler)
  - flame_smoke: 0.2 (minimal smoke)
  - flame_vorticity: 1.0
  - burning_rate: 1.2 (faster burn = quicker wisps)
  - noise_scale from preset
- Cache directory: set_cache_directory(domain, "luminous_fire_wisps")
- Flow source: body_obj itself (add Fluid modifier with FLOW type, FIRE, INFLOW)
  - fuel_amount: 0.8 (less than standalone fire 1.5 -- wisps not inferno)
  - temperature: 1.5 (lower = subtler)
  - surface_distance: 0.5 (close to body surface)
- Material on domain: Principled Volume with Blackbody (same pattern as fire_cinema_template)
  - Density: 5.0
  - Blackbody Intensity: 1.2
  - Blackbody Tint: warm gold (1.0, 0.9, 0.7, 1.0)
- Material name: efs_lumi_wisp_material
- NOTE: body_obj gets BOTH a Fluid Flow modifier (for fire wisps) AND a particle system (for particles). Blender supports multiple modifiers on the same object.
- Returns (domain_obj, flow_obj) tuple where flow_obj is body_obj with flow modifier

_create_corona(body_obj):
- Creates efs_lumi_corona: a duplicate of the body mesh, slightly scaled up (1.05x)
- Material: Emission shader with Fresnel-based edge detection:
  - Node setup: Fresnel (IOR=1.5) -> ColorRamp (sharp falloff: black at 0.3, white at 0.7) -> multiply with Emission Strength
  - Base Emission Strength: 3.0 (keyframeable by treble)
  - Emission Color: bright cyan-white (0.9, 0.95, 1.0)
  - Material Surface output (not Volume) -- this is a surface glow effect
  - Alpha Blend transparency mode so only edges are visible
- Material name: efs_lumi_corona_material
- The corona mesh follows body_obj shape keys (if body has shape keys, corona should be parented or copy shape keys)
  - Simplest approach: parent corona to body_obj, apply Copy Location + Copy Rotation constraints
  - For shape key sync: use a Mesh Deform or Shrinkwrap modifier binding corona to body
- Keyframeable: Emission Strength (treble frequency)
- Returns the corona object

_setup_compositor(layer_configs):
- Uses compositor_layers.create_layer_collection() to organize objects:
  - "LumiVolFill": [efs_lumi_fill]
  - "LumiParticles": [efs_lumi_body] (body emits particles -- needs to be in particle collection)
  - "LumiFireWisps": [efs_lumi_fire_domain] (domain renders the fire volume)
  - "LumiCorona": [efs_lumi_corona]
- Uses compositor_layers.setup_multi_layer() with:
  - {"name": "VolFill", "collection": "LumiVolFill", "opacity": 1.0, "bloom": True, "bloom_threshold": 0.6}
  - {"name": "Particles", "collection": "LumiParticles", "opacity": 0.9}
  - {"name": "FireWisps", "collection": "LumiFireWisps", "opacity": 1.0, "bloom": True, "bloom_threshold": 0.7}
  - {"name": "Corona", "collection": "LumiCorona", "opacity": 0.8, "bloom": True, "bloom_threshold": 0.5}
- Background-to-foreground order: VolFill (deepest) -> Particles -> FireWisps -> Corona (outermost)

_create_camera():
- efs_lumi_camera at (4, -4, 1.5) -- front-facing at person height
- efs_lumi_target empty at (0, 0, 0.9) -- center of body
- Track To constraint
- 50mm lens (portrait framing)
- DOF f/2.8, focus distance matching target
- Returns camera object

_create_lighting():
- Minimal lighting (the being IS the light):
- One subtle rim light (efs_lumi_rim_light) behind the body at (-2, 3, 3), 30W cool blue (0.7, 0.8, 1.0) -- just enough to provide depth separation when effects are dim
- That is ALL. No key light. No fill light. The volumetric fill, fire, and corona ARE the illumination.

PUBLIC FUNCTIONS (four-function API):

create_luminous_scene(mask_dir, quality="preview", particle_mode="flame"):
- Entry point. Loads quality preset, calls all internal functions in order:
  1. save_before_operate("luminous_being_scene")
  2. _setup_scene(preset)
  3. body = _create_body(mask_dir, quality)
  4. fill = _create_volumetric_fill(body)
  5. particles = _create_particle_system(body, particle_mode)
  6. fire_domain, fire_flow = _create_fire_wisps(body, preset)
  7. corona = _create_corona(body)
  8. camera = _create_camera()
  9. rim_light = _create_lighting()
  10. _setup_compositor(...)
- Returns JSON summary dict (same pattern as fire_cinema_template)

apply_luminous_audio(audio_json_path, preset_name="luminous_being"):
- Wrapper around keyframe_generator.apply_audio_keyframes() with luminous_being preset
- Same pattern as fire_cinema_template.apply_audio()

bake_luminous(clean_first=True):
- Bakes the Mantaflow fire wisps domain (efs_lumi_fire_domain)
- Uses async_bake.start_bake() same as fire_cinema_template.bake_fire()

render_luminous(output_name="luminous_being", frame=None):
- Renders via async_render same pattern as fire_cinema_template.render_fire()
- Single frame or full animation
